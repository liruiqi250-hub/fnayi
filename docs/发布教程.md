# 发布新版本教程（小白版）

> 看完本文大约需要 5 分钟，以后每次发版只需 1 分钟。

---

## 准备工作

你已经有了 GitHub 账号，接下来：

### 1. 创建仓库

打开 https://github.com/new

- **Repository name** 填：ile-translator（或你喜欢的名字）
- 其他都不用动，点 **Create repository**

### 2. 把你的代码上传到仓库

打开你电脑上的项目文件夹，在地址栏输入 cmd 回车，然后一条一条执行以下命令：

`ash
git init
git add .
git commit -m "v1.1.1 初始版本"
git branch -M main
git remote add origin https://github.com/liruiqi250-hub/fnayi.git
git push -u origin main
`

> 把 liruiqi250-hub 换成你注册时的用户名。
> 执行到最后一步会弹出浏览器让你登录 GitHub，登录即可。

---

## 发新版本（核心步骤）

以后每次你想发布新版本，只需要以下 **4 步**：

### 第 1 步：打包新安装包

在项目文件夹打开终端（PowerShell），执行：

`powershell
python -m PyInstaller FileTranslator.spec --noconfirm
`

打包完成后，把 .env.example 复制进去：

`powershell
Copy-Item .env.example dist\FileTranslator\
`

然后用 Inno Setup 打包安装包（或者手动执行 scripts\build_installer.ps1）。

打包好的安装包在 outputs\ 文件夹里，文件名类似 FileTranslator_Setup_v1.1.1.exe。

### 第 2 步：在 GitHub 上发布 Release

打开浏览器，访问：

`
https://github.com/liruiqi250-hub/fnayi/releases
`

点绿色的 **Create a new release** 按钮。

### 第 3 步：填写 Release 信息

按照下图填写：

| 字段 | 填什么 | 示例 |
|------|--------|------|
| **Tag version** |  + 版本号 | 1.2.0 |
| **Release title** | 版本名称 | 1.2.0 - 新增文本翻译功能 |
| **Write** 框 | 更新内容（给用户看的） | 见下方模板 |

**更新内容模板：**

`
## v1.2.0 更新内容

### 🆕 新功能
- 新增文本翻译功能，支持粘贴文字直接翻译
- 新增深色/浅色主题切换
- 新增自动更新检测

### 🛠 优化
- 文件整理 UI 重做，去掉多余的翻译选项
- 翻译引擎增加 VPN 使用提醒

### 🐛 Bug 修复
- 修复自定义大模型引擎无法使用的问题
- 修复暗黑模式下弹窗文字看不清的问题
`

### 第 4 步：上传安装包 + 发布

- 把桌面上的 FileTranslator_Setup_v1.2.0.exe **拖到下方的框里**（Attach binaries）
- 检查没问题后，点绿色的 **Publish release** 按钮

**搞定！** 用户打开软件 → 设置 → 检查更新，就能看到新版本并下载了。

## 常见问题

**Q：版本号怎么定？**
A：每次发版加 1。当前是 1.1.1，下次 1.1.2（小修改），再下次 1.2.0（大功能），再下次 1.2.1。

**Q：Tag version 要加 v 吗？**
A：要加，写成 1.2.0。软件会自动去掉 v 来对比版本号。

**Q：用户检查更新时显示「仓库不存在」？**
A：打开软件 → 设置 → 通用设置，检查 GitHub 仓库填的是不是 liruiqi250-hub/fnayi，有没有写错。

**Q：安装包太大传不上去？**
A：GitHub 单个文件限制 2GB，安装包通常 100MB 左右，完全够用。
